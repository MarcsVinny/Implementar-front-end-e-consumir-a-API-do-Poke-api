# PokéDex - Consumindo PokéAPI

Este é um projeto de uma PokéDex completa, desenvolvida como parte de uma atividade acadêmica para implementar um front-end funcional consumindo a API pública [PokeAPI](https://pokeapi.co/).

## 🚀 Funcionalidades

- **Listagem de Pokémons**: Renderização de 20 Pokémons por vez.
- **Paginação**: Navegação entre as páginas da API (20 Pokémons por página).
- **Busca**: Pesquisa de Pokémons por nome com exibição de card individual.
- **Detalhes**: Visualização detalhada (Nome, Imagem, Tipos, Altura e Peso) ao clicar em um card.
- **Design Moderno**: Interface responsiva e estilizada com base em conceitos do Figma, incluindo cores dinâmicas por tipo de Pokémon.

## 🛠️ Tecnologias Utilizadas

O projeto foi construído utilizando apenas tecnologias web puras (Vanilla), sem o uso de frameworks ou bibliotecas externas, conforme solicitado:

- **HTML5**: Estrutura semântica da aplicação.
- **CSS3**: Estilização moderna, Grid Layout, Flexbox e design responsivo.
- **JavaScript (ES6+)**: Consumo de API via `fetch()`, manipulação assíncrona do DOM e lógica de paginação/busca.

## 📂 Estrutura do Projeto

O projeto é composto por apenas 3 arquivos principais para manter a simplicidade e organização:

```text
/
├── index.html    # Estrutura da página
├── style.css     # Estilização e design
└── script.js     # Lógica e consumo da API
```

## 🎨 Design

O design foi inspirado no layout do Figma fornecido, focando em:
- Cards organizados em grid.
- Bordas coloridas dinamicamente de acordo com o tipo principal do Pokémon.
- Modal de detalhes para uma experiência de usuário limpa.
- Responsividade básica para dispositivos móveis.
